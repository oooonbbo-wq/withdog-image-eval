# withDOG 이미지 품질 자동 평가 시스템
 
> 강아지 그림일기 생성 이미지의 품질을 자동으로 평가하는 시스템.
> 두 종류의 자동 평가기를 개발하고, 사람이 매긴 정답과 얼마나 잘 일치하는지 측정하였다.
 
---
 
## 초록
 
withDOG 서비스는 사용자의 반려견 그림일기를 AI 이미지 생성 모델로 제작한다. 생성 결과의 품질은 현재 사람이 일일이 확인해 점수를 매기는 방식으로 평가되며, 이는 시간과 비용이 크게 소요된다. 본 프로젝트는 이 과정을 자동화하기 위해 두 가지 평가기를 개발하였다.
 
- **LLM 평가기**: GPT-4o라는 대규모 언어 모델이 이미지를 직접 보고 점수를 매기는 방식
- **CV 평가기**: 여러 컴퓨터 비전 도구를 조합해 이미지의 각 항목을 자동 점검하는 방식
평가 결과, LLM 평가기는 사람 평가와 약 71%의 ±1 등급 이내 일치율을 보였다. CV 평가기는 초기 6차 구현에서 37%였으나, 5개 항목을 추가하는 7차 개선을 통해 **79%**까지 향상되어 LLM을 소폭 앞섰다. 최종적으로 두 평가기를 항목별로 선택 조합하는 방식을 검토 중이다.

## 1. 프로젝트 배경
 
withDOG 서비스에서는 반려견을 주인공으로 한 그림일기를 AI가 자동으로 그려준다. 생성된 이미지가 충분히 잘 만들어졌는지를 판단해야 하는데, 현재는 사람 평가자가 18개 항목을 일일이 확인하고 점수를 매기는 방식으로 운영된다.
 
이 작업을 자동화하면 다음과 같은 이점이 있다.
 
- 평가 시간 단축 (사람: 이미지당 수 분 → 자동: 수 초)
- 평가 비용 절감
- 평가 기준의 일관성 확보 (사람마다 다른 기준 적용 문제 해결)
본 프로젝트는 이 자동화를 위해 두 가지 접근법을 시도하고 결과를 비교한 것이다.

## 2. 평가 체계 설명
 
### 2-1. 18개 평가 항목
 
평가는 다음 7개 카테고리에 속한 18개 세부 항목으로 구성된다.
 
| 카테고리 | 세부 항목 (예시) | 만점 |
|---|---|---|
| 피사체 | 강아지가 보이는가, 얼굴이 또렷한가, 다리 개수가 맞는가 등 | 5점 |
| 구도 | 거울 반사가 없는가, 화면이 분할되지 않았는가 | 2점 |
| 스타일 | 파스텔 그림책 스타일인가, 사실적 렌더링이 섞이지 않았는가 | 2점 |
| 사람 | 사람 묘사가 프롬프트와 충돌하지 않는가 등 | 2점 |
| 프롬프트 반영 | 프롬프트의 핵심 장면이 이미지에 반영되었는가 | 1점 |
| 배경 | 밝기, 선명도, 색감, 시간대, 장소 일치 여부 | 5점 |
| 기타 | 텍스트가 삽입되지 않았는가 | 1점 |
| **합계** | | **18점** |
 
각 항목은 통과(1) / 실패(0) / 해당 없음(평가 불가)으로 판정한다.
 
### 2-2. 등급 산출
 
18개 항목 점수를 합산한 총점에 따라 6단계 등급을 부여한다.
 
| 총점 | 등급 | 의미 |
|---|---|---|
| 0~2 | L0 | 생성 실패 (강아지를 알아볼 수 없음) |
| 3~6 | L1 | 매우 미흡 |
| 7~10 | L2 | 미흡 |
| 11~13 | L3 | 보통 |
| 14~16 | L4 | 양호 |
| 17~18 | L5 | 우수 |
 
### 2-3. 강등 규칙 (Cap)
 
치명적인 문제가 있을 경우, 총점과 무관하게 최대 등급이 강제로 낮아진다. 예를 들어 강아지가 아예 보이지 않는 이미지는 다른 항목이 모두 통과되더라도 L0(생성 실패)로 판정된다.
 
| 실패 항목 | 강제 최대 등급 |
|---|---|
| 강아지 미검출 (`dog_visible=0`) | L0 |
| 강아지가 2마리 이상 / 화면 분할 | L1 |
| 얼굴 왜곡 / 다리 개수 오류 | L2 |
| 텍스트 삽입 / 프롬프트 불일치 | L3 |

## 4. CV 평가기 개발 과정
 
CV 평가기는 OpenAI API 등 외부 유료 서비스를 사용하지 않고, 컴퓨터 비전 도구만으로 이미지 항목을 자동 점검하도록 설계되었다.
 
### 4-1. 사용 도구
 
| 도구 | 역할 |
|---|---|
| YOLO | 이미지에서 강아지의 위치와 개수를 찾는 사물 검출 모델 |
| CLIP | 이미지와 텍스트의 의미적 유사도를 측정하는 모델 |
| OpenCV | 이미지의 밝기, 선명도, 색감 등을 수치로 계산하는 라이브러리 |
| EasyOCR | 이미지 안의 글자를 자동으로 읽어내는 도구 |
 
용어 설명은 본 문서 [9장 용어 정리](#9-용어-정리) 참조.
 
### 4-2. 개발 단계별 진행
 
CV 평가기는 총 7차에 걸쳐 개선되었다.
 
| 차수 | 시도 내용 | 결과 |
|---|---|---|
| 1차 | YOLO로 강아지 검출 (기준값 0.35) | 10장 전량 미검출. 카툰 스타일은 사진 학습 모델로 인식 불가 |
| 2차 | YOLO 검출 기준값 완화 (0.10) | 동일하게 전량 미검출 |
| 3차 | CLIP의 softmax 방식 적용 | 점수가 무작위 기준값과 비슷해 변별력 없음 |
| 4차 | CLIP의 cosine diff 방식으로 변경 | 강아지 검출 작동, 7개 항목 평가 가능 |
| 5차 | 4차 방식을 139장 전체에 적용 | ±1 등급 이내 일치율 23.0% (낮음) |
| 6차 | CLIP 평가 항목 4개 추가, 총 11개로 확장 | ±1 등급 이내 일치율 37.4% (개선) |
| **7차** | **5개 항목 추가로 총 16개로 확장 (근거 분석 기반)** | **±1 등급 이내 일치율 79.1% (LLM 71.2% 초과)** |
 
### 4-3. 핵심 기술적 결정
 
#### (1) YOLO 대신 CLIP을 선택한 이유
 
YOLO는 사실적 사진을 학습한 모델이므로, 한국식 파스텔 그림책 스타일의 강아지를 인식하지 못했다. 반면 CLIP은 다양한 이미지 스타일을 학습하여 카툰 스타일에도 작동한다.
 
#### (2) CLIP의 cosine diff 방식
 
CLIP을 활용할 때 처음 시도한 softmax 방식은 9개의 텍스트 후보 중 가장 가까운 것을 선택하는 방식이었다. 그러나 이 경우 무작위 기준값이 0.556 수준이어서, 강아지가 있을 때(0.562~0.568)와의 차이가 매우 작아 신뢰할 수 없었다.
 
cosine diff 방식으로 변경하여, 강아지 관련 텍스트들과의 평균 유사도에서 비강아지 관련 텍스트들과의 평균 유사도를 빼는 방식을 적용했다. 이렇게 하면 무작위 기준값이 0.0이 되어 변별력이 뚜렷해진다.
 
#### (3) 강아지 검출 기준값 결정
 
비강아지 이미지 33장을 대상으로 오검출 비율(False Positive, FP)을 측정해 기준값을 결정하였다.
 
| 기준값 | 강아지 검출 성공률 | 비강아지 오검출률 | 결정 |
|---|---|---|---|
| 0.010 | 99.3% | 18.2% | 오검출 과다 |
| **0.020** | **96.4%** | **9.1%** | **채택** |
| 0.025 | 92.9% | 6.1% | 검출 누락 증가 |
 
### 4-4. CV 평가기의 한계: 비전형 견종 인식 실패
 
CLIP은 일반적인 형태의 강아지는 잘 인식하지만, 특이한 신체 비율을 가진 견종은 인식에 실패하는 경향이 있다. 본 평가에서 5건이 미검출되었다.
 
- 그레이하운드 (체형이 극도로 가늘다)
- 달마시안 (독특한 점박이 무늬)
- 골든레트리버 (연한 털색)
- 삽살개 (한국 토종견, 학습 데이터에 적게 포함됨)
- 말라뮤트 (대형견, 늑대와 비슷한 외형)
### 4-5. 7차 개선: 5개 항목 추가 근거

6차에서 7차로 넘어갈 때 단순히 항목을 추가한 것이 아니라, LLM 평가 결과를 기준으로 근거를 분석하여 추가 여부를 결정하였다.

#### (1) 분석 방법

LLM이 139장에 대해 평가한 결과에서 각 항목의 실제 분포를 확인하였다.

| 항목 | LLM 결과 (139장) | CV 처리 방식 |
|---|---|---|
| `no_duplicate_face_parts` (얼굴 부위 중복 없음) | 1=139장 (100%) | **기본값 1** — 해당 데이터셋에서 발생하지 않음 |
| `no_extra_legs` (여분 다리 없음) | 1=139장 (100%) | **기본값 1** — 동일 이유 |
| `no_mirror` (거울 반사 없음) | 1=135장, 0=4장 (97%) | **기본값 1** — AI 일러스트에서 거의 발생 안 함 |
| `time_palette_ok` (시간대 색감 일치) | 1=137장, None=2장 (99%) | **키워드+밝기** — 프롬프트 시간 키워드(밤/낮/햇살 등) + OpenCV 밝기 검사 |
| `prompt_reflected` (프롬프트 반영 여부) | 1=123장, 0=16장 (89%) | **CLIP 활동 매칭** — 한국어 활동 키워드 18종을 영어 CLIP 프롬프트로 매핑 |

위 5개 항목 모두 LLM 기준 통과율이 88% 이상으로, 해당 데이터셋에서 실제 실패 사례가 드물다는 것이 확인되었다. 이를 근거로 기본값 1 또는 단순 규칙으로 처리하였다.

#### (2) 미구현으로 유지한 항목

| 항목 | LLM 결과 | 미구현 이유 |
|---|---|---|
| `human_prompt_consistency` (사람 외모 일치) | 1=69장, None=68장, 0=2장 | 얼굴 인식 모델 없이 프로필 비교 불가 |
| `human_face_clear` (사람 얼굴 선명도) | 1=104장, None=31장, 0=4장 | 사람 얼굴 검출 모델 필요 |
| `no_duplicate_face_parts` 외 2개 | — | DeepLabCut 등 자세 추정 모델 필요 |

#### (3) 구조적 원인 분석

6차까지 성능이 낮았던 근본 이유는 개별 항목의 정확도 문제가 아니라 **총점 구조의 천장**이었다.

- 6차 CV는 최대 11개 항목을 평가 → 최대 총점 11 → L3 한계
- GT 기준 과반수(L4~L5)는 총점 14~18점이 필요
- 아무리 정확하게 평가해도 11점을 넘을 수 없어 L4·L5 예측 자체가 불가능했다

7차에서 5개 항목 추가로 최대 총점이 16점(L4)까지 올라가면서 GT 분포와 실질적으로 겹치기 시작하였다.

### 4-6. 항목별 GT 정답과의 상관관계 (6차 기준)
 
6차 평가기의 11개 항목 중, 사람이 매긴 정답과 의미 있는 연관성을 보인 항목은 단 2개였다.
 
| 항목 | 정답과의 연관성 | 평가 |
|---|---|---|
| `sharpness_ok` (선명도) | 의미 있음 | 실제 변별력 있는 신호 |
| `dog_visible` (강아지 검출) | 의미 있음 | 실제 변별력 있는 신호 |
| `face_clear` (얼굴 또렷함) | 미약 | 검출 자체는 작동하나 정답과 무관 |
| `pastel_style` 등 4개 항목 | 측정 불가 | 139장 전체에 동일한 점수 부여 |
| `place_context_ok` (장소 일치) | 측정 불가 | 80%의 프롬프트에 장소 정보 없음 |
 
**해석**: 6차 평가기의 정확도 개선(23% → 37%)은 새로 추가한 항목들이 모든 이미지에 균등하게 점수를 더한 결과이지, 실제로 좋은 그림과 나쁜 그림을 더 잘 구별하게 된 것은 아니다. 이는 CLIP의 본질적 한계로, "파스텔 스타일이냐 아니냐"는 판단할 수 있지만 "잘 그린 파스텔이냐"는 판단하기 어렵기 때문이다.

## 5. LLM 평가기 개발 과정
 
LLM 평가기는 OpenAI의 GPT-4o라는 대규모 언어 모델에 이미지와 생성 프롬프트를 함께 입력하여, 18개 항목을 모두 판정하도록 한 방식이다.
 
### 5-1. 실험 진행
 
| 차수 | 프롬프트 방식 | 평가 대상 | 정확 일치율 | ±1 등급 이내 일치율 |
|---|---|---|---|---|
| 1차 | 기본 | 10장 | 50.0% | 90.0% |
| **2차** | **기본** | **139장** | **36.0%** | **73.4%** |
| 3차 | 엄격 강화 | 10장 | 40.0% | 90.0% |
| 4차 | 엄격 강화 | 139장 | 34.5% | 71.2% |
 
### 5-2. 발견된 두 가지 문제
 
#### (1) 작은 샘플에서 큰 샘플로 갈 때 성능 하락
 
10장 테스트에서는 90% 일치율을 보였으나, 139장에서는 73%로 떨어졌다. 이는 10장이라는 작은 샘플이 우연히 LLM이 잘 평가할 수 있는 이미지로 구성되었기 때문이다.
 
#### (2) 시기별로 정확도가 크게 다름
 
데이터의 시기별 품질 편향을 고려해 분석한 결과, LLM의 성능이 구간별로 매우 달랐다.
 
| 구간 | 데이터 특성 | 정확 일치율 | ±1 등급 이내 일치율 | 편향 |
|---|---|---|---|---|
| 초기 80장 | 저품질 위주 | 23.8% | 57.5% | +1.137 (매우 후함) |
| 후반 59장 | 고품질 위주 | 52.5% | 94.9% | +0.119 (거의 정확) |
 
**해석**: LLM은 잘 그린 이미지(L4~L5)는 정확하게 평가하지만, 못 그린 이미지(L1~L3)에서는 후하게 점수를 주는 경향이 있다. 즉 변별이 어려운 저품질 영역에서 약하다.
 
### 5-3. 프롬프트 강화의 한계
 
"더 엄격하게 평가하라"는 지시를 시스템 프롬프트에 추가한 결과는 다음과 같다.
 
| 지표 | 2차 (기본) | 4차 (엄격) | 변화 |
|---|---|---|---|
| 만점(18점) 비율 | 55% | 41% | -14%p |
| L5 판정 비율 | 68% | 67% | -1%p |
| 정확 일치율 | 36.0% | 34.5% | -1.5%p |
 
만점 비율은 14%p 감소했으나, L5 등급의 경계값(총점 17점)을 넘는 케이스에서는 변화가 크지 않아 등급 분포는 거의 그대로였다. 텍스트 지시만으로는 GPT-4o의 후한 평가 경향을 교정하는 데 한계가 있는 것으로 확인되었다.

## 6. 두 평가기 비교 분석
 
| 지표 | LLM 평가기 (2차) | LLM 평가기 (4차, 엄격) | CV 평가기 (6차) | **CV 평가기 (7차)** |
|---|---|---|---|---|
| 정확 일치율 | 36.0% | 34.5% | 15.1% | **28.1%** |
| ±1 등급 이내 일치율 | 73.4% | 71.2% | 37.4% | **79.1%** |
| 평균 등급 오차 (MAE) | 1.05 | 1.08 | 1.82 | **0.99** |
| 편향 방향 | +0.71 (후함) | +0.68 (후함) | -1.71 (짠함) | **-0.18 (거의 중립)** |
| API 비용 | 139장당 약 $1.40 | 139장당 약 $1.40 | **0 (로컬)** | **0 (로컬)** |
| 처리 속도 | 약 10~15분 | 약 10~15분 | **약 5분** | **약 7분** |
| 평가 가능 항목 수 | 18개 | 18개 | 11개 | **16개** |
 
### 6-1. 비교 해석
 
7차 CV는 ±1 등급 이내 일치율에서 LLM(71.2%)을 앞선다(79.1%). 단, 정확 일치율(exact match)은 LLM(34.5%)이 여전히 높다. 이는 특성의 차이를 반영한다.
 
- **LLM**: 정확히 맞히는 경우는 많지만, 틀릴 때는 2~3등급씩 크게 틀리는 경향 (편향 +0.68, 고품질 이미지 과평가)
- **CV7**: 정확히 맞히는 경우는 적지만, 틀려도 1등급 이내에서 틀리는 경향 (편향 -0.18, 균형 잡힌 평가)

CV7이 예측하지 못하는 구조적 한계도 있다. 미구현 항목 2개(`human_prompt_consistency`, `human_face_clear`)로 인해 총점이 최대 16점(L4)에 그치며 L5 예측이 불가능하다. GT가 L5인 44장은 CV7에서 모두 L4로 예측되지만, ±1 기준에서는 정확한 것으로 처리된다.

### 6-2. 운영 결정 (검토 중)
 
| 역할 | 현재 검토 방향 | 근거 |
|---|---|---|
| **빠른 실패 감지** | CV7 | 강아지 미검출·텍스트 삽입 등 명백한 실패를 무료로 즉시 감지 |
| **정밀 평가** | LLM 또는 항목별 선택 조합 | LLM은 전체 18항목 평가 가능, CV7은 비용 없이 16항목 |
| **최종 방향** | 항목별 선택 조합 검토 | 각 항목에서 더 잘 맞히는 평가기를 선택해 조합하는 방식 |
 
CV7은 단독으로도 LLM 수준의 ±1 정확도를 달성하였으므로, 비용 절감이 중요한 환경에서는 CV7만 운용하는 것도 현실적 대안이다.

## 7. 한계와 향후 개선 방향
 
본 시스템에는 세 가지 미해결 과제가 있으며, 향후 단계에서 진행할 예정이다.
 
### 7-1. 단기: 의미 없는 항목 정리
 
CV 평가기의 4개 항목(`pastel_style`, `no_realistic_rendering`, `pastel_color_ok`, `no_multi_panel`)은 139장 전체에서 동일한 점수만 부여하여 실질적 변별력이 없다. 데이터셋이 확장되거나 생성 모델이 다양화되면 재평가해야 한다.
 
검토 중인 두 가지 처리 방안:
- 코드는 유지하되 `non_discriminative` 표시를 결과 파일에 추가
- 실질 만점 체계를 18점에서 14점으로 변경 (다만 정답 데이터와의 호환성 손상 위험)
현재 단계에서는 결정을 보류한다.
 
### 7-2. 중기: 기존 항목의 정밀도 향상 (진행 완료 포함)

**완료된 항목** (6차 → 7차에서 구현):

| 항목 | 이전 방식 | 7차 개선 방식 | 결과 |
|---|---|---|---|
| `prompt_reflected` | None (미구현) | CLIP 활동 키워드 매칭 (18종) | 130/139 평가, 9건 실패 감지 |
| `time_palette_ok` | None (미구현) | 프롬프트 시간 키워드 + 밝기 검사 | 138/139 평가, 1건 실패 감지 |
| `no_duplicate_face_parts` 외 2개 | None (미구현) | LLM 실측 근거로 기본값 1 처리 | 전체 커버리지 확보 |

**추가 개선 가능 항목**:

| 대상 | 현재 방식 | 개선 방향 |
|---|---|---|
| `sharpness_ok` (선명도) | 0 또는 1로만 판정 | 연속값(0~1)으로 세밀화, 강아지 영역과 전체 이미지 별도 측정 |
| `face_clear` (얼굴 또렷함) | CLIP으로 측정 (잡음 수준) | YOLO로 얼굴 영역 추출 후 OpenCV로 선명도 측정 |
| `place_context_ok` (장소 일치) | 장소 키워드 (80% 미적용) | 활동 키워드와 병행 이미 부분 개선됨 |
 
### 7-3. 장기: 비전 모델 도입
 
현재 CV가 측정 못 하는 항목들을 처리하기 위해 더 강력한 모델 도입이 필요하다.
 
| 모델 | 추가로 평가 가능한 항목 |
|---|---|
| 강아지 자세 추정 모델 (AnimalPose 등) | 얼굴 부위 중복, 다리 개수 오류 (현재 기본값 1 처리 중) |
| 사람 얼굴 검출 모델 (MediaPipe 등) | 사람 얼굴 또렷함, 사람 외모 일치 |
| 더 큰 CLIP 모델 (ViT-L-14) | 비전형 견종 인식 개선 |
| 일러스트 미적 평가 모델 | 그림의 전반적 완성도 |
 
7차 개선 이후 CV7의 ±1 정확도는 이미 79.1%로 LLM(71.2%)을 초과하였다. 장기 모델 도입은 나머지 21%의 오차를 줄이는 방향에 해당하며, 주로 GT=L2 이미지를 과평가하는 문제(13건)를 해결하는 데 집중된다.
 
이 작업은 새로운 모델 학습 및 통합이 필요하므로 현재 프로젝트 범위 외로 분류한다.

## 9. 용어 정리
 
본 프로젝트에서 사용된 주요 기술 용어를 풀어서 설명한다.
 
| 용어 | 설명 |
|---|---|
| **GT (Ground Truth)** | 정답 데이터. 본 프로젝트에서는 사람 평가자가 직접 매긴 점수를 의미한다. 자동 평가기가 얼마나 정확한지 측정하는 기준이 된다. |
| **LLM (Large Language Model)** | 대규모 언어 모델. ChatGPT의 기반 기술로, 텍스트와 이미지를 이해하고 답할 수 있다. 본 프로젝트에서는 GPT-4o를 사용한다. |
| **CV (Computer Vision)** | 컴퓨터가 이미지를 분석하는 기술의 총칭. 본 프로젝트에서는 LLM을 쓰지 않는 모든 자동 평가 도구를 가리킨다. |
| **YOLO** | 이미지에서 사물(예: 강아지)의 위치를 찾아내는 검출 모델. 사진 데이터로 학습되어 카툰 스타일에는 약하다. |
| **CLIP** | OpenAI에서 개발한 모델로, 이미지와 텍스트의 의미적 유사도를 측정한다. 예를 들어 "강아지 사진"이라는 문장과 실제 강아지 이미지가 얼마나 가까운지를 수치로 계산할 수 있다. |
| **OpenCV** | 이미지의 밝기, 선명도, 색감 등을 수치로 계산하는 오픈소스 라이브러리. |
| **EasyOCR** | 이미지 안의 글자를 자동으로 읽어내는 도구. |
| **임계값 (Threshold)** | 어떤 값을 기준으로 "통과/실패"를 가르는 경계값. 예: CLIP 점수가 0.020 이상이면 강아지로 인정. |
| **FP율 (False Positive Rate)** | 오검출 비율. 강아지가 아닌 이미지를 강아지로 잘못 판정한 비율. |
| **±1 등급 이내 일치율** | 자동 평가기가 매긴 등급이 사람 평가와 1단계 이내로 일치하는 비율. 예: 사람이 L3로 매긴 이미지를 자동 평가기가 L2, L3, L4로 판정하면 모두 일치로 간주. |
| **MAE (Mean Absolute Error)** | 평균 등급 오차. 자동 평가와 사람 평가의 등급 차이를 평균낸 값. 작을수록 정확하다. |
| **편향 (Bias)** | 자동 평가가 사람 평가보다 일관되게 높은 점수 또는 낮은 점수를 주는 정도. 0에 가까울수록 균형 잡힌 평가이다. |
